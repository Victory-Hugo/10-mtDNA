# 要求

我需要你构建**生物信息学pipeline**，该pipeline需要具备良好的可扩展性和易用性。目前，暂定实现如下需求：

- **输入**：以`/mnt/f/onedrive/文档（科研）/脚本/Download/10-mtDNA/2-单倍群层级划分/input/ID_Hap.tsv`为例。是一个tsv（也可能是csv）文件。第一列是ID，第二列是单倍群名称（可能有表头、可能没有表头）。
- 处理：首先需要对输入的单倍群的名称按照``Haplogrep单倍群分型订正表.tsv``进行纠正，以确保输入的单倍群名称是标准的。然后，根据`phylotree_build17.csv`中的单倍群层级关系，对输入的单倍群进行层级划分。最后，根据用户选择的目标文件（`目标_LLT.tsv`或`目标_YuChunLi.tsv`）和对应的纠错文件（`错误纠正_LLT.tsv`或`错误纠正_YuChunLi.tsv`），将层级划分结果转换为用户需要的宏单倍群名称。
- **输出**：
  - `tsv`文件，包含了输入文件中的ID、单倍群名称(原始的、phylotree的、YuChunLi的、LLT的)、phylotree的单倍群级别、phylotree的单倍群父节点名称、phylotree的单倍群定义变异信息。
