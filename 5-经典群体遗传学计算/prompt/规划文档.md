# Scikit-allel pipeline 规划文档

## 项目目标

建立一个标准化流程，用于从人类 mtDNA/chrY （单倍体）变异数据中计算**遗传多样性参数**，并输出可用于群体遗传分析的结果。

本流程适用于：

1. 单群体多样性分析
2. 多群体分化分析

本流程优先支持以下指标：

1. 核苷酸多样性 π
2. Watterson’s θ
3. Tajima’s D
4. 单倍型多样性
5. 群体间核苷酸分化 dxy
6. 群体间 Fst，优先 Hudson Fst

这些指标均可由 scikit-allel 直接支持，或在其数组与计数框架上稳定实现。

## 分析的特殊性

scikit-allel 文档指出，单倍体调用应使用 HaplotypeArray，而不是常规 GenotypeArray。GenotypeArray 适用于 ploidy 大于 1 的调用。

## 总体架构

推荐采用四层结构。

### 输入层
输入文件包括：

1. bgzip 压缩且带索引的 VCF（如果未压缩，提示用户先压缩并索引）
2. 样本分组表（TSV/CSV），至少包含 ID 和 Group 两列（代码自动检查是否有表头，默认第一列为ID，第二列为 Group）


### 存储层

先将 VCF 转为 Zarr，再进行统计。scikit-allel 提供 vcf_to_zarr()，支持按字段选择、样本子集和区域抽取，适合建立可复用中间层。

### 统计层

围绕以下对象组织：

HaplotypeArray

AlleleCountsArray

population metadata


### 输出层

统一输出为表格文件，建议 TSV 。每个表必须具备标准列名，方便合并与追踪。



## 配置文件设计

推荐使用 `YAML`。

```yaml
project_name: human_diversity
input_vcf: data/data.vcf.gz
sample_table: meta/samples.tsv
output_dir: output
contig_name: chrM

conda_env: BigLin

windows:
  enable: true
  size: 500
  step: 250

metrics:
  within_groups:
    - pi
    - theta_w
    - tajima_d
    - haplotype_diversity
  between_groups:
    - dxy
    - hudson_fst

runtime:
  n_threads: 4
  random_seed: 42
```

## 计算步骤设计

对每个群体分别计算：

1. π
2. θw
3. Tajima’s D
4. haplotype diversity

其中：

sequence_diversity() 用于 π
watterson_theta() 用于 θw
tajima_d() 用于 Tajima’s D
haplotype_diversity() 用于单倍型多样性

## 计算群体间统计

对每一对群体计算：

1. dx
2. Hudson Fst

其中：

sequence_divergence() 用于 dxy
windowed_hudson_fst() 或 hudson_fst() 用于 Fst

scikit-allel 文档指出，sequence_divergence() 与 sequence_diversity() 都支持 is_accessible 参数，适合处理可分析位点分母问题。

## 滑窗统计

**若开启**窗口分析，按配置文件中的窗口大小和步长执行：

1. windowed_diversity
2. windowed_watterson_theta
3. windowed_tajima_d
4. windowed_hudson_fst
