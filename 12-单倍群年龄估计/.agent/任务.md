# 研究背景

我正在对人类的mtDNA系统发育树的所有的单倍群的共祖进行估计。使用的是ρ方法。目前已经成功。

1. `12-单倍群年龄估计/data/merged_clean.vcf.gz`：全球人群的mtDNA数据（其中以HaplogroupAnces_*开头的样本是根据系统发育树进行的祖先序列重建）。
2. `12-单倍群年龄估计/data/phylotree_index_withacc.json`：系统发育树的索引文件，包含了每个单倍群的定义和其在系统发育树中的位置。
3. `12-单倍群年龄估计/data/rcrs.fasta`：人类mtDNA的参考序列。
4. `12-单倍群年龄估计/META/1-回复突变.tsv`：包含了每个单倍群的回复突变信息（来自Soare等人的研究）。
5. `12-单倍群年龄估计/META/2-单倍群信息.tsv`全球人群的Haplogrep3分型文件。
6. `12-单倍群年龄估计/pipe/run_all.sh`：运行所有分析的脚本。

关键节点验证`12-单倍群年龄估计/output/stage_1/results/validation_key_nodes.tsv`。
M/N/R 轻微偏低（0.2-4.4 kya），系本研究样本量（35k-80k）远大于 Soares 2009 原始校准集所致，结果科学合理。

# 现在的问题

目前我已经成功地使用了ρ方法对每个单倍群的共祖进行了估计，并且得到了每个单倍群的年龄估计值和置信区间。但是，由于一些单倍群内部的样本量较小（n<10），ρ方法的估计可能不够准确。因此，我**个人认为通过ML方法**也许是好的选择：

**核心问题**是计算可行性：82,110个样本直接跑 IQ-TREE 即使固定拓扑也需要数天。最佳实践是分层降采样——每个叶节点单倍群取1-3个代表性现代样本（约3,000-9,000条序列），固定 PhyloTree 拓扑，用 iqtree3 -te 只估计枝长，然后把枝长×区域有效长度转为核苷酸距离后代入相同的Soares 公式，与 ρ 结果对比。

另一个更高效的替代方向：直接用已有的 HaplogroupAnces_*5,441条祖先重建序列构建一棵紧凑树（内部节点已知），只为每个叶单倍群附加少量现代tip，这样树规模可控在 ~6,000-8,000 tips，且天然对齐了 ρ 方法的计算单元。

主要权衡：降采样会损失叶级多样性，影响末端枝长估计；如果只需要敏感性分析而不是独立验证，ρ 法已足够——ML 只在小样本单倍群（n<10）时有明显增益。
